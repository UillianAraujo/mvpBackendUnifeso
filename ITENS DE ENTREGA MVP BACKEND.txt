ESCOPO
É oque está dentro dos limites do projeto, é tudo oque o MVP vai entregar. FORA DO ESCOPO são funcionalidades que não vou incluir nesse momento, mas sim futuramente.

REQUISITOS FUNCIONAIS
São aquelas ações que o sistema precisa executar, ex: cadastrar, buscar, deletar, atualizar...

REQUISITOS NÃO-FUNCIONAIS
Características que diz respeita a qualidade do sistema, ex: segurança, desempenho, escalabilidade, tempo de resposta...

EX.: 
o sistema deve responder em até 2 segundos (não-funcional)

o sistema deve permitir o login de usuários (funcional)

QUANDO FOR APONTAR O REQUISITOS FUNCIONAIS E NÃO-FUNCIONAIS, TEM QUE MOSTRAR NÃO SÓ O QUE O SISTEMA FAZ, MAS COMO ELE SE COMPORTA.
